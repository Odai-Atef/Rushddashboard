## Summary

Intercept HTTP 401 (Unauthorized) responses from all API calls in the React application. Clear stored auth tokens, append the current page as a `redirect` query parameter, and navigate the user to `/auth/login` with an Arabic session-expired message. Ensure graceful handling of concurrent 401s, avoid redirect loops, and allow in-flight requests to complete before redirecting.

## Implementation Strategy

### High-Level Flow

1. **ApiClient intercept**: In `requestWithRetry()`, when `executeRequest()` throws an error with `statusCode === 401`:
   - If `isRedirecting` is already true, re-throw the error silently (no duplicate redirect)
   - Otherwise, set `isRedirecting = true`, call `clearAuthToken()`, build redirect URL with `expired=true` and current path as `redirect`, then navigate to `/auth/login`

2. **LoginPage display**: Read `?expired` from `URLSearchParams`. If present, show the Arabic message banner above the form. Read `?redirect` and store it in local state for post-login navigation.

3. **Post-login redirect**: After successful login, instead of hardcoded `navigate('/dashboard')`, navigate to the `redirect` param value (default `/dashboard`).

### Files to Modify

| File | Changes |
|------|---------|
| `src/api/client.ts` | Add module-level `isRedirecting` flag; intercept 401 in `requestWithRetry`; call `clearAuthToken`; construct and execute redirect URL |
| `src/app/components/LoginPage.tsx` | Import `useSearchParams`; read `expired` and `redirect` params; display Arabic session-expired banner; use dynamic redirect after login |

### Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| Infinite redirect loop if login page itself triggers 401 | Guard: skip redirect if current path is `/auth/login` |
| Race condition with multiple parallel 401s | Module-level `isRedirecting` flag ensures only one redirect executes |
| In-flight form submission interrupted mid-request | 401 is already non-retryable; fetch promise resolves/rejects before redirect happens |
| Open redirect via manipulated `?redirect` param | Validate `redirect` is a relative path starting with `/` before navigating |
