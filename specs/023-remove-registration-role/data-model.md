# Data Model: Remove Role Selection from Registration Flow

## Entity: Self-Registration Data

Represents the information collected and transmitted during user-initiated registration.

| Attribute | Type | Required | Constraints |
|-----------|------|----------|-------------|
| email | string | Yes | Valid email format |
| password | string | Yes | Minimum 6 characters |
| firstName | string | Yes | Non-empty, max 100 chars |
| lastName | string | Yes | Non-empty, max 100 chars |
| companyId | UUID | Yes | Valid UUID v4 |

**Removed**: `roleId` is no longer collected or sent during self-registration.

## Entity: Default Role

The role automatically assigned by the backend when a user self-registers.

| Attribute | Type | Constraints |
|-----------|------|-------------|
| id | UUID | System-defined default role identifier |
| name | string | Display name (e.g., "User", "Employee") |

## Entity: User Account

The record created upon successful registration.

| Attribute | Type | Source |
|-----------|------|--------|
| id | UUID | Generated by backend |
| email | string | From registration payload |
| firstName | string | From registration payload |
| lastName | string | From registration payload |
| companyId | UUID | From registration payload |
| roleId | UUID | Assigned automatically by backend (default role) |

## Relationships

- **Self-Registration Data** → submitted to **Backend Registration Endpoint**
- **Backend** → assigns **Default Role** → to new **User Account**
- **User Account** → belongs to one **Company** (via `companyId`)

## Validation Rules

- All fields in Self-Registration Data are required.
- Password must be at least 6 characters.
- Email must match a valid email format.
- `companyId` must be a valid UUID.
- No extra fields (including `roleId`) should be present in the frontend payload.
- Backend ignores any stray `roleId` if present and assigns the configured default role.
